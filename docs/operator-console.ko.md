## 화면 구성


## 라이브 배선 (시뮬레이션 → 실백엔드)

콘솔의 데이터 레이어는 브라우저 내 시뮬레이션이 아니라 **실백엔드**를 소비합니다:

- **WebSocket** `/ws/v1/events` → `onDepth` · `onResp` · `onRec` · `onSysErr` 핸들러가 각 메시지를 렌더 함수로 어댑트
- **REST**: 세션 시작 / 큐·중단·수동·기침 액션 / 승인 / 감사 조회·검증
- 백엔드 상태 이름·게이팅은 [게이팅 문서](depth-and-gating.md)와 1:1 대응 → 화면 상태 전이 = 백엔드 실제 전이

### 표면 깊이 vs 두께

라이브 신호에서 `z`는 **절대 표면 거리**(카메라~흉부, ≈800mm)이고 두께는 `estimated_thickness_mm`(≈240mm)로 별개 값입니다. 그래서:

- 파형 Y축은 고정 기준선이 아니라 **신호 자체의 중앙값**에 자동 정렬(고정값이면 트레이스가 화면 밖으로 나감)
- 신호 패널은 표면 깊이(z)와 추정 두께를 각각 표시
- LUT 하이라이트는 **두께**로 매칭

### 추천 de-dupe

백엔드는 안정 유지 중 매 프레임 추천을 재발행하고 `recommendation_id`가 증가합니다. 콘솔은 `(manual|kVp/mAs)` 시그니처가 바뀔 때만 다시 렌더·이벤트 기록해 스팸을 막고, `lastRecommendationId`는 항상 최신으로 갱신해 승인 대상이 유효하게 유지됩니다.

## 파형 렌더 (Canvas 2D)

`waveBuf`에 최근 260 샘플을 링버퍼로 유지하고, 상태별 색으로 선분을 그립니다(안정=녹색, 불안정/타임아웃=주황, abort=적색). 안정 구간은 배경 하이라이트, 마지막 점은 상태색 도트. DPR(devicePixelRatio)에 맞춰 캔버스를 스케일해 고해상도에서 선명합니다.

## 국제화 (i18n)

- 한/영 사전(`I18N`) + `data-i18n` / `data-i18n-html` / `data-i18n-ph` / `data-i18n-title` 속성
- 언어 선택은 `localStorage`에 저장, 재방문 시 복원
- 함수형 문자열 지원(예: `sub_stable:d => …${d}ms…`)으로 동적 문구도 번역

## 설정 페이지

- **서버 연결**: 게이트웨이 주소(host:port)를 편집·저장(localStorage) → `/health`·`/state`로 실시간 상태·서비스·기기 정보 표시. 다른 엣지 장비로 콘솔을 재지정하면 WS/REST 원본이 즉시 재연결.
- **카메라**: `/devices` 열거 → 시리얼별 선택·연결 (자세히는 [카메라 추상화](camera-abstraction.md))
- **보정 / 기기 정보**: 빈 침대 보정, 백엔드 실측 상태(연결·보정·세션)

## 단축키

검사(study) 화면에서만 동작(Esc 제외). 입력 필드 포커스 중엔 차단:

| 키 | 동작 |
|---|---|
| `S` | 세션 시작 / 종료 |
| `B` | 호흡 정지 큐 |
| `C` | 기침 유발 |
| `A` | 추천 승인 |
| `Esc` | 워크리스트로 (설정 열렸으면 설정 닫기) |
